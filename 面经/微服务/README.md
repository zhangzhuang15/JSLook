### 微前端特点
1. 技术无感 
不要求固定使用某一种前端框架及其全家桶

2. 独立开发，独立部署
微前端下的主应用、子应用都是可以独立开发、部署、维护的，之前没有耦合

3. 增量更新
本意是只用更新做出修改的那部分，没有修改的部分不需要做更新。
放在微前端上来讲，就是说如果有一个旧系统，你一下子重构所有的技术栈，非常困难。但是，你可以将旧系统拆解成若干个部分，没个部分都是子应用，那么你就可以一个部分一个部分用全新技术栈重构，没有重构的部分，依旧使用以前的技术栈在线上运行

4. 运行时隔离
主应用和子应用的数据是相互隔离的。
